# Octoparse LIPS Scraping Portfolio

## 概要
LIPS（コスメ口コミサイト）において、
**最新投稿一覧 → 投稿詳細 → ユーザープロフィール → SNSリンク取得**
までを一気通貫で自動化したスクレイピング設計の実績です。

単純な一覧取得ではなく、
「取得漏れ」「重複」「遷移失敗」が発生しやすい構造を前提に、
**安定稼働を最優先した設計**を行っています。

---

## 対象サイトの特性
- 投稿数が多く、最新一覧が高速で更新される
- 一覧・詳細・プロフィールがすべて別DOM構造
- 同一URLでも表示内容が動的に変化するケースあり
- SNSリンクは「存在しない場合はHTML自体に存在しない」

---

## 実現したこと
- 最新投稿一覧ページから **複数投稿をループ処理**
- 投稿詳細ページで **投稿日・投稿URL・ユーザーURLを取得**
- ユーザープロフィールページへ遷移し、
  - Instagram
  - X（旧Twitter）
  - YouTube
  - TikTok
  - Webサイト  
  の **登録されているもののみ取得**
- 未登録SNSは **空欄として正しく処理**
- 重複データが発生しない設計
- Googleスプレッドシートへの後続連携を前提としたデータ構造

---

## 設計上のポイント
- クリック遷移とURL遷移を使い分け
- ページ遷移後の **待機時間（Delay）を明示的に制御**
- 取得対象要素を限定したXPath設計
- 「取得できない」＝「空欄が正解」とする判断基準の明確化
